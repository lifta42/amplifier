Detailed Description

A simple implementation of determine finite automata.

A DFA is built up with several states and a transition function. The function accepts a state and a symbol as its arguments, and returns another state.

After building, a DFA instance is frozen and may work any times you want. For each time it runs, the DFA accepts a series of labels as input and transitions with it. The state of DFA can be traced.

Additional callbacks may be set for states. Proper callback will be triggered if the corresponded state is reached.
